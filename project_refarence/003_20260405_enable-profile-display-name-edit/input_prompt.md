# input_prompt

## ユーザー依頼の原文
プルフィール編集画面、表示名変更も有効にしたい。これはfirebase認証などに関係ないよね？

## この作業で扱う対象
- 表示名変更と Firebase 認証同期の関係整理
- 表示名変更が Firebase 同期で上書きされないようにするバックエンド修正

## 前提条件
- プロフィール編集APIで表示名更新機能は既に存在する
- 認証時に Firebase トークン情報で users を upsert している

## 制約事項
- 既存の認証・プロフィール機能を壊さない
- 変更理由をテストで担保する

## 実施前の整理メモ
- 現状は upsertFromFirebase が既存ユーザーの display_name を更新している可能性がある
- ユーザーがアプリ内で変更した表示名を優先すべき

## 実施方針
1. 現行ロジック確認
2. 既存ユーザーの表示名を Firebase 同期で上書きしない修正
3. ユニットテスト追加
4. テスト実行と report 反映
