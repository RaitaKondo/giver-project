# input_prompt

## ユーザー依頼の原文
プロフィール編集でアカウント写真を変更できるようにしたい。ユーザーが写真をフォルダから選択する。投稿の写真と同じようにGCSを使う予定です。写真の選択は一枚。容量も規定します。
何が必要か教えてください
実行してください

## この作業で扱う対象
- プロフィール編集のアカウント写真アップロード機能の実装（フロント/バックエンド）
- GCS保存、枚数1枚制限、容量制限、形式制限

## 前提条件
- 既存の投稿画像アップロードは `multipart/form-data` + GCS保存済み
- 既存プロフィール更新は JSON の `photoUrl` 更新方式
- 認証は既存の `/api/me/*` を利用

## 制約事項
- 既存実装との整合性を優先
- 画像は1枚のみ
- 容量上限は明示し、フロント/バック双方で検証

## 実施前の整理メモ
- DBスキーマ変更を最小化し、既存 `users.photo_url` の利用を維持する
- 互換性のため既存プロフィール更新APIは残し、新規アップロードAPIを追加する
- レスポンスの `photoUrl` は既存の利用箇所に影響しない形を維持する

## 実施方針
1. バックエンドにプロフィール画像アップロードAPIを追加
2. GCS保存サービスへプロフィール画像保存メソッドを追加
3. ユーザーサービスで写真更新処理を追加
4. フロントで単一ファイル選択UIと送信APIを追加
5. バックエンド単体テストを追加して動作を検証
